Tyrant's Monocle
What Does it Do?
This application is intended to build prediction models for VainGlory matches. Our server queries the VainGlory API to gather statistics about users and matches which it uses to build a predictive model based on a rudimentary Naive Bayes classifier implementation. Wining matches train the predictive model for many of the statistics which the VainGlory API provides.

There are several benefits to this approach:

  • It's learning as it goes
  • Very efficient processing
  • Avoiding bulk API scraping

Direction
The prediction algorithm for this application could likely be improved using additional machine-learning concepts. For example, the k-Nearest Neighbors algorithm could be employed as a strategy to classify matches which can be represented as N-dimensional vectors in N-dimensional hyperspace. However this approach may need to consider several different spaces. For example, it may increase accuracy to consider an 'Actor' space as well as a 'Player' space and a 'Team' space.

This approach will be challenging for several reasons:

  • kNN algorithm requires a distance calculation which could be very challenging to represent on discrete variables, such as Actor
  • This approach will be much more expensive in terms of computation, as larger parts of the dataset will have to be brought into memory
Stack

Front

This application is developed in Angular and built with Node/Gulp using the boilerplate project angularjs-gulp-browserify-boilterplate (link). This project's repository is hosted on GitHub @ mtaylor22/Tyrants--Monocle (link).

Back

The backend application is developed in Node using Hapi, Mongodb, and Redis. The Bull (link) package is used to ensure API requests are not exceeding the limit. This project is hosted on GitHub @ mtaylor22/gold-miner (link)
